IP

ip地址：IntelAddress

  • 唯一定位一台网络上的计算机

  • 127.0.0.1 : 本机localhost

  • ip分类：

    • ipv4/ipv6
      • IPV4 : 127.0.0.1, 四个字节（0-255）组成,45亿（北美独占30亿）
      • IPV6 : 2001:0bb2:aaaa:0015:0000:0000:1aaa:1312,八个无符号整数(128)
    • 公网（互联网）-私网（局域网）
      • 192.168.xx.xx,局域网，专门给组织机构使用

  • 域名：记忆ip

端口

端口表示计算机上的一个程序的进程，举一个形象的比喻：一栋楼就是一个ip地址，一栋楼中的每一户人家的门就是一个个的端口号。

  • 不同的进程有不同的端口号，用来区分软件

  • 电脑的端口号被规定数量是0～65535（理想化）

  • TCP/UDP ：65535*2，单个协议下，端口号不能重复（比如TCP用了一个80端口，那么UDP就不能再用了）

  • 端口分类

    • 公有端口：0~1023

      • HTTP：80
      • HTTPS：443
      • FTP：21
      • Telent ：23

    • 程序注册端口：1024-49151（分配给用户或者程序）

      • Tomcat：8080
      • MySQL：3306
      • Oracle：1521

    • 动态、私有：49152~65535

import java.net.InetSocketAddress;

public class hello {
    public static void main(String[] args) {
        InetSocketAddress a = new InetSocketAddress("127.0.0.1",8080);
        System.out.println(a);
        System.out.println(a.getAddress());//返回地址+端口
        System.out.println(a.getHostName());//返回地址
        System.out.println(a.getPort()); //返回端口
    }
}

至此，也算是学完了基于TCP/UDP协议的通信，现在就来总结一下：

UDP:

  • 发送方

    • 1.通过端口创建一个DatagramSocket连接
    • 2.创建一个BufferedReader对象来接收InputStreamReader对象里面的输入
    • 3.读取信息，并把它转换成字节流
    • 4.创建一个DatagramPacket对象的包
    • 5.发送包

    但是总结之后，无非就是两步：开放端口，发送包

  • 接收方

    • 1.通过端口创建一个DatagramSocket连接
    • 2.创建一个DatagramSocket包，用来接受发送端发送过来的信息
    • 3.通过getData获取包的数据，然后解析成字符串String，并打印出来
